# 공유 레시피 (recipes/shared/) — 커뮤니티 테스트 케이스

이 폴더는 **의도적으로 공유하는(정제된) 레시피**만 담습니다. 여기 있는 것만 git 으로 추적되고,
`recipes/` 루트에 **자동 저장**되는 레시피는 무시됩니다(개인 크롤 URL·이력 보호).

## 왜 분리하나

레시피 CSV 의 `meta` 에는 크롤 당시의 **실제 URL(`url`)** 과 매칭 힌트로 쓰인 **스크랩 스니펫(`example`)** 이
들어갑니다. 자동 저장분을 그대로 공개하면 내 검색어·관심사가 그대로 노출됩니다. 그래서 공유는
**손수 정제한 사본**만, 이 폴더에 둡니다.

## 공유용으로 정제하는 법 (커밋 전)

1. `recipes/<자동저장>.csv` 를 이 폴더로 복사하고, 사이트를 알아볼 이름으로 바꿉니다.
   예) `채용사이트_공고목록.csv`, `쇼핑몰_상품상세_체인.csv`
2. `meta` 행에서 개인정보를 마스킹합니다:
   - `url` → 검색어/식별 파라미터를 지운 **대표 URL**(예: `https://www.example.com/search?searchword=EXAMPLE`)
   - 체인 레시피의 `clean_url` 도 동일하게.
3. `field` 행의 `example`(힌트 스니펫)은 **비우거나 일반 예시**로 교체합니다.
   - 레시피는 구조 경로/셀렉터로 동작하므로, example 이 비어도 추출은 됩니다(LLM 치유 힌트만 약해짐).
4. 커밋. 다른 사람은 이 파일을 `recipes/` 루트로 복사하면 같은 사이트에서 바로 재현할 수 있습니다.

## 형식

`core/schema.py` 의 CSV 레시피 형식(`kind,name,tag,cls,attr,path,example` — meta 행 + field 행)을 따릅니다.
사람이 열어봐도 읽히도록 설계돼 있습니다.
